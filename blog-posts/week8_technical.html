An Introduction to Big O

Oct 19th, 2014

Big O is a function notation which is used in computer science to describe in simple terms how the growth rate of an algorithm affects its efficiency (for example in terms of time or space it needs to complete).

Understand Big O by example

If you read the Wikipedia article Big O notation you may feel overwhelmed and rather lost if you are not familiar with several concepts in mathematics.
In this article I will try to give a basic high level understanding of how to read Big O notation and make sense of it without needing to go into too many details. Let's start with some code: 

def find_in_arr(value, arr)
  arr.each { |el| return true if value == el }
  false
end

find_in_arr('B', ['A', 'B', 'C']) # => true
find_in_arr('X', ['A', 'B', 'C']) # => false

In the above method we return true if wee can find our value in the array otherwise we return false. This one of the simplest algorithm we can think of and it's great to understand how Big O works.
Now let's consider for a moment how long the 2 different calls to the method take to compute. It's clear that the first call where we look for 'B' will compute faster since the method can return at the second iteration.
The second call to find_in_arr takes longer because it will iterate through all elements without finding a match.

One important aspect to understand in Big O is that it always takes into account the worst case scenario, which in mathematics is referred to as upper bound.
Finally in terms of our find_in_arr method we can say that, as the array increases in size the upper bound of the time it requires to compute is proportional to the size. Using Big O you would write it as O(n)
Let's look at one more example, a bit of a silly one: 

def am_i_true?(obj)
  !!obj
end

a = "hello"
b = false

am_i_true? a  # => true
am_i_true? b  # => false

We can see that here passing in whatever obj doesn't really make much difference to the method since it's only task is to return the boolean value of the obj. To describe this method in terms of Big O you would write O(1) which means this is a straight line. It is a constant.

Another interesting example often used to clarify the Big O notation is nested iteration for example: 

def arr_prod(arr)
  result = []
  arr.each do |outer|
    arr.each do |inner|
      result << [outer, inner]
    end
  end
  result
end

arr_prod ['x', 'y']  # => [["x", "x"], ["x", "y"], ["y", "x"], ["y", "y"]]

In this case we try to create a product of each element in an array with all other elements.
When the array increases in size, the method will not only have to iterate through more elements but each additional element it will have to iterate once again through the entire array. This is the perfect example to represent exponential growth which in Big O is represented by O(n^2) or O(n²)

This concludes a basic introduction to Big O. If you want to learn more I suggest having a look at the Wikipedia article Big O notation as well as this great chart that plots various common Big O functins
