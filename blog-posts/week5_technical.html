Class in Ruby what is it and how to use it

Sep 25th, 2014

Today I would like to briefly introduce the concept of Class in ruby.
Ruby is one very cool language, that has incredible flexibility and is so clear. To understand classes in Ruby let's consider for a moment how ruby is organized.

Everything is an Object

The most important and perhaps famous statement for ruby is "everything is an object". Ruby is a so called an object oriented programming language. This means that it is organized around the concept of objects, trying to replicate the object of the physical / meta-physical world we experience. The focus around its organization is on the objects (actors) rather than their actions.
Let's say that we are building a program to keep track of our friends and relationship between each other, here is a basic example of how we could do that: 

class User

  attr_accessor :name, :birthdate, :friends

  def initialize(name, birthdate, *friends)
    @name = name
    @birthdate = Date.parse birthdate
    @friends = friends
  end

  def say_my_name
    puts "Hello there, my name is #{name}"
  end

  def to_s
    puts "#{name}, born: #{birthdate}"
  end

end

Class as a blueprint

A class is a blueprint for objects to come to life. Following the example above we are creating the mold for a user giving her a name, birthdate and a container to store and retrieve friends. Let's see this in action: 

sara = User.new 'sara', '1970-01-01'
=> #<User:0x007f836ba3e530 @birthdate=#<Date: 1970-01-01 ((2440588j,0s,0n),+0s,2299161j)>, @friends=[], @name="sara">
scarlet = User.new 'scarlet', '1972-02-02', sara
=> #<User:0x007f836b9d5d00
 @birthdate=#<Date: 1972-02-02 ((2441350j,0s,0n),+0s,2299161j)>,
 @friends=
  [#<User:0x007f836ba3e530 @birthdate=#<Date: 1970-01-01 ((2440588j,0s,0n),+0s,2299161j)>, @friends=[], @name="sara">],
 @name="scarlet">

First we created a new user 'sara', passing her the name and brithdate, then we created another user 'scarlet' and this time we passed also 'sara' and the cool thing about this is that we can see that when scarlet is instantiated (created) sara appears in scarlet's list of friends.

Methods, the object actions

Let's now have a look at methods, which in ruby are those lines of code enclosed between a def .. end like: 

def some_method
  #some code here
end

Methods are the actions of the objects, they determine what the object can do. In this case let's have a look at a few more examples: 

sara.to_s
sara, born: 1970-01-01

sara.say_my_name
hello there, my name is sara

I typed sara.to_s and we got:
'sara, born: 1970-01-01'
Isn't that cool? we can define actions and values that our objects manipulate
