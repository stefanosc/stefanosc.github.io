JavaScript - a Brief Overview of Pros and Cons

Sep 13th, 2014

JavaScript has experienced a tremendous growth in the last decade projecting it as one of the fastest growing and most popular programming languages used by developers, let's briefly review some of the aspects that brought JavaScript to be so popular.

JavaScript Pros


Speed - client side vs server side

JavaScript is executed on the client side, meaning the code is not processed on the server. In this way there is no connectivity lag or any other lag that may be introduced by the server and connectivity to it. It just all happens on the user side

Versatility - any browser can interpret it

JavaScript makes for a super versatile language. Because practically any browser can interpret this language it can be integrated in any web application or website making it a great complement to any web project regardless of the core language used.

Ease of Use

JavaScript is a simple language, the learning curve is less steep compared to other languages making it an easy choice for many developers.
Of course this doesn't mean JavaScript can't be used to create complex web applications and in fact quite the opposite. A multitude of frameworks have emerged in recent years to providing great solution to build feature rich web applications, some popular frameworks are Ember.js and Angular

JavaScript Cons


Security issues

Because JavaScript is processed on the client side it is more susceptible to malicious attacks.
Malicious snippets of JavaScript could compromise the security of the user's computer.

Inconsistency of implementations

Despite JavaScript great popularity, it's implementation among different browsers historically has not been standard, and although many efforts have been made to standardize it as a language, cross browser compatibility is still an issue.
This is why frameworks like jQuery were introduced.
